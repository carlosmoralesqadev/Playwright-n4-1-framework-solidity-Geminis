TABLA 01

Jerarquía de Selectores

¿cuál usar primero?
# Selector Cuándo usarlo Ejemplo listo para copiar
1 ★ getByRole()
← PRIMERA OPCIÓN		 Botones, links, inputs, headings, checkboxes. Cualquier elemento con rol ARIA claro. page.getByRole('button', { name: /login/i })
page.getByRole('textbox', { name: /usuario/i }).fill('admin')
page.getByRole('link', { name: /click here/i }).click()
page.getByRole('checkbox').nth(0).click()
page.getByRole('heading', { name: /título/i, level: 2 })
2 getByLabel() Input con un <label> asociado en el HTML. page.getByLabel('Usuario').fill('admin')
page.getByLabel('Contraseña').fill('1234')
3 getByText() Validar mensajes, párrafos, texto visible. Cuando el elemento no tiene rol claro. expect(page.getByText('You logged in')).toBeVisible()
page.getByText('Eliminar', { exact: true }).click()
4 getByPlaceholder() Inputs con texto gris pero sin label. "El dev fue perezoso." page.getByPlaceholder('Buscar...').fill('prueba')
page.getByPlaceholder('Email').fill('test@test.com')
5 getByTestId()
← PACTO QA-DEV		 HTML tiene atributo data-testid="valor". Ideal cuando dev lo agrega. page.getByTestId('boton-guardar').click()
page.getByTestId('input-email').fill('x@x.com')
6 locator()
← ÚLTIMO RECURSO		 IDs o Clases CSS. También para encapsular búsquedas en un contenedor. page.locator('#id-unico').click()
page.locator('#input-example input').fill('texto')
page.locator('.inventory_item').filter({ hasText: 'Backpack' })
REGLA MENTAL: Probá en orden 1 → 2 → 3 → 4 → 5 → 6. El primero que funcione, ese usás.
TIP REGEX: Siempre preferí /texto/i en lugar de 'texto' → ignora mayúsculas y espacios extra.
¿Tiene rol ARIA? (button, link, textbox, checkbox, heading, img...)
├── SÍ → getByRole('rol', { name: /texto/i })
└── NO →
¿Tiene <label>? → getByLabel('texto')
¿Tiene texto visible? → getByText('texto')
¿Tiene placeholder? → getByPlaceholder('texto')
¿Tiene data-testid? → getByTestId('id')
¿Tiene ID o clase CSS? → locator('#id') / locator('.clase')
¿Muchos iguales? → locator('.contenedor').filter({ hasText: 'X' }).getByRole(...)
TABLA 02

Roles ARIA + ¿de dónde sale el { name }?

HTML → rol + nombre accesible
Grupo HTML Rol Playwright Ejemplo completo
TÍTULOS <h1> / <h2> / ... / <h6> 'heading' getByRole('heading', { name: /mi título/i, level: 2 })
INTERACTIVOS <button> 'button' getByRole('button', { name: /enviar/i })
<a href="..."> 'link' getByRole('link', { name: /ir al inicio/i })
<input type="checkbox"> 'checkbox' getByRole('checkbox') ← rara vez lleva name
<input type="radio"> 'radio' getByRole('radio', { name: /opción a/i })
<select> 'combobox' getByRole('combobox', { name: /país/i })
INPUTS / FORMS <input type="text"> 'textbox' getByRole('textbox', { name: /usuario/i })
<input type="password"> 'textbox' getByRole('textbox', { name: /contraseña/i })
<textarea> 'textbox' getByRole('textbox', { name: /comentario/i })
<form> 'form' getByRole('form')
ESTRUCTURA <img> 'img' getByRole('img', { name: /user avatar/i })
<ul> o <ol> 'list' getByRole('list')
<li> 'listitem' getByRole('listitem')
<table> 'table' getByRole('table')
⚠ Si el elemento es <div> o <span> sin rol → getByRole NO funciona. Pasá a getByText(), getByTestId() o locator('.clase').
¿De dónde sale el { name }? — buscalo en este orden en el inspector
ℹ El { name } NO viene del atributo name="..." del HTML. Viene del nombre accesible del elemento.
Prioridad Fuente HTML de ejemplo name que escribís
1 ★ mejor Texto interno <button>Enviar</button> { name: /enviar/i }
2 <label> asociado <label>Usuario</label><input id="u"> { name: /usuario/i }
3 aria-label <button aria-label="Cerrar sesión"><i/></button> { name: /cerrar sesión/i }
4 placeholder <input placeholder="Escribe tu email"> { name: /escribe tu email/i }
5 alt (solo imgs) <img alt="User Avatar"> { name: /user avatar/i }
Cuándo no lleva { name }
// Checkbox sin label visible getByRole('checkbox').nth(0) // Único elemento de ese tipo getByRole('button') ← funciona igual
⚠ Trampa — toHaveAttribute con objeto
// ❌ INCORRECTO — falso positivo .toHaveAttribute('href', { name: 'http://url.com' }) // ✅ CORRECTO — string exacto .toHaveAttribute('href', 'http://url.com/') // ✅ CON REGEX — búsqueda parcial .toHaveAttribute('href', /elementalselenium/)
TABLA 03

Posición y Encapsulamiento

.first() .last() .nth() + Padre→Hijo
Filtro Qué hace Ejemplo
.first() Primer elemento de la lista page.getByRole('button', { name: /delete/i }).first().click()
.last() Último elemento de la lista page.getByRole('button', { name: /delete/i }).last().click()
.nth(0) Primero (índice empieza en 0) page.getByRole('checkbox').nth(0).click()
.nth(1) Segundo page.getByRole('checkbox').nth(1).click()
.nth(2) Tercero page.getByRole('button', { name: /delete/i }).nth(2).click()
REGLA: nth(0) = first(). nth(último) = last().
REGLA: Al borrar .nth(1), el que era el tercero pasa a ser el segundo → el siguiente .last() lo captura.
Patrón Encapsulamiento — Padre → Hijo
💡 Problema: Hay 10 botones "Add to cart" → Playwright lanza error de ambigüedad.
Solución: Paso 1 → localizá el contenedor único. Paso 2 → buscá el elemento DENTRO.
// ❌ Ambiguo — hay 10 botones iguales await page.getByRole('button', { name: /add to cart/i }).click() // ERROR // ✅ Correcto — primero el contenedor, luego el botón dentro const producto = page.locator('.inventory_item').filter({ hasText: 'Sauce Labs Backpack' }) await producto.getByRole('button', { name: /add to cart/i }).click()
Caso típicoCódigo
Agregar 3 elementos de un click page.getByRole('button', { name: /add/i }).click({ clickCount: 3 })
Validar que hay exactamente 3 expect(page.getByRole('button', { name: /delete/i })).toHaveCount(3)
Borrar el segundo page.getByRole('button', { name: /delete/i }).nth(1).click()
Borrar el último que quedó page.getByRole('button', { name: /delete/i }).last().click()
Botón dentro de fila de tabla const fila = page.getByRole('row').filter({ hasText: 'Carlos' })
await fila.getByRole('button', { name: /editar/i }).click()
TABLA 04

Aserciones expect()

qué validan + trampas + auto-waiting
Aserción Qué valida Ejemplo
toBeVisible() Visible en pantalla await expect(locator).toBeVisible()
not.toBeVisible() NO es visible await expect(locator).not.toBeVisible()
toHaveText('texto') Texto idéntico — más estricto await expect(locator).toHaveText('You entered: T')
toContainText('texto') Contiene esa cadena — parcial await expect(locator).toContainText('logged in')
toBeChecked() Checkbox/radio marcado await expect(locator).toBeChecked()
not.toBeChecked() Checkbox/radio NO marcado await expect(locator).not.toBeChecked()
toBeDisabled() Elemento deshabilitado await expect(locator).toBeDisabled()
toBeEnabled() Habilitado e interactuable await expect(locator).toBeEnabled()
toBeEnabled({ timeout: 10000 }) Idem, con espera extendida await expect(locator).toBeEnabled({ timeout: 10000 })
toHaveCount(n) Lista tiene exactamente n elementos con auto-retry await expect(locator).toHaveCount(3)
toHaveAttribute('attr', 'val') Atributo tiene ese valor exacto await expect(locator).toHaveAttribute('target', '_blank')
toHaveAttribute('attr', /regex/) Atributo contiene el patrón await expect(locator).toHaveAttribute('href', /selenium/)
⚠ Trampa — count() sin auto-retry
// ❌ INCORRECTO — sin auto-retry expect(locator).count() === 0 // Si el DOM tarda → falla aunque luego sea correcto // ✅ CORRECTO — con auto-retry await expect(locator).toHaveCount(0)
💡 Auto-Waiting — cómo funciona
// Playwright espera hasta 5 seg por defecto // No necesitás waitForTimeout() ni page.pause() // Si el server es lento → extendé el tiempo .toBeEnabled({ timeout: 10000 }) // 10 seg
Métodos de acción — referencia rápida
.click()Clic simple
.click({ clickCount: 3 })3 clics en una sola línea
.fill('texto')Escribe directo (como pegar), sin simular teclado
.pressSequentially('TEXTO')Simula tecla por tecla (activa key listeners del DOM)
.press('Escape')Tecla especial: Enter, Escape, Tab, Backspace, ArrowDown...
.hover()Mueve el mouse encima (revela tooltips y menús ocultos)
.selectOption('valor')Selecciona una opción de un <select>
page.goto('https://...')Navega a la URL
TABLA 05

Texto Exacto vs Regex

cuándo usar cada uno
Tipo Sintaxis Encuentra No encuentra
Parcial (default) getByText('Log') Login, Logout, Logging —
Exacto getByText('Login', { exact: true }) Solo 'Login' 'Login Now', ' Login '
Regex { name: /login/ } login, Login, LOGIN —
Regex /i ★ { name: /login/i } Login, LOGIN, login, lOgIn —
REGLA: En { name } de getByRole → SIEMPRE usá /regex/i. Evita errores por espacios o mayúsculas.
REGLA: { exact: true } solo aplica en getByText() y getByLabel(), NO dentro de { name } de getByRole.
Caso prácticoCódigo listo para copiar
Botón con texto visible page.getByRole('button', { name: /enviar/i })
Input con label page.getByRole('textbox', { name: /usuario/i })
Link con texto parcial page.getByRole('link', { name: /elemental/i })
Validar texto exacto en pantalla await expect(locator).toHaveText('You entered: ESCAPE')
Validar que contiene parte del texto await expect(locator).toContainText('logged in')
Atributo con URL exacta await expect(locator).toHaveAttribute('href', 'http://url.com/')
Atributo con URL parcial await expect(locator).toHaveAttribute('href', /elementalselenium/)
Exacto en getByText page.getByText('Eliminar', { exact: true })
Distinguir "Log" de "Login" page.getByText('Log', { exact: true }) ← solo encuentra 'Log'
⚠
Trampa del falso positivo:
toHaveAttribute('href', { name: 'http://url.com', exact: true })
El test puede PASAR aunque la URL sea diferente — solo verifica que el atributo exista.

Prueba de fuego: Cambiá la URL dentro del objeto por 'https://google.com'. Si el test sigue pasando → está mal escrito. Usá string o regex.
