1. AL INICIAR UNA NUEVA CARPETA DE PROYECTO (PLAYWRIGHT)
-----------------------------------------------------------
# Entra en la carpeta de tu nuevo ejercicio y ejecuta:
git init

# CONFIGURA TU IDENTIDAD LOCAL (Evita usar la de otras cuentas)
git config user.name "carlosmoralesqadev"
git config user.email "carlosmoralesqadev@gmail.com"

# Verifica que se guardó bien para este proyecto:
git config user.name
git config user.email


2. CONECTAR CON GITHUB (SOLO LA PRIMERA VEZ POR REPO)
-----------------------------------------------------------
# Crea el repo en la web de GitHub (SIN ESPACIOS en el nombre).
# Luego, enlaza tu carpeta local con la nube:
git remote add origin https://github.com/carlosmoralesqadev/nombre-del-repo.git

# Cambia el nombre de la rama a 'main' (estándar actual):
git branch -M main

# Sube los archivos por primera vez y conecta las ramas:
git push -u origin main


3. CICLO DE TRABAJO DIARIO (CADA VEZ QUE TERMINES UN TEST)
-----------------------------------------------------------
# 1. Mira qué archivos cambiaste:
git status

# 2. Prepara todos los archivos para la "foto":
git add .

# 3. Guarda los cambios con un mensaje descriptivo:
git commit -m "Mensaje explicando qué testeaste"

# 4. Sube los cambios a GitHub:
git push


4. COMANDOS DE CONSULTA (PARA SABER DÓNDE ESTÁS)
-----------------------------------------------------------
# Ver el historial de cambios corto:
git log --oneline

# Ver a qué repositorio estás apuntando:
git remote -v

# Ver qué líneas exactas cambiaste en el código:
git diff
