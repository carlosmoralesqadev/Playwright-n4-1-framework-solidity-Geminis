 COMPARATIVA — string, exact y regex
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 TIPO               │ SINTAXIS                               │ ENCUENTRA              │ NO ENCUENTRA
────────────────────┼────────────────────────────────────────┼────────────────────────┼────────────────────────
 Parcial (default)  │ getByText('Log')                       │ Login, Logout, Logging │ —
 Exacto             │ getByText('Login', { exact: true })    │ Solo 'Login'           │ 'Login Now', ' Login '
 Regex parcial      │ { name: /login/ }                      │ login, Login, LOGIN    │ —  (pero no ignora ñ)
 Regex con /i       │ { name: /login/i }                     │ Login, LOGIN, login    │ —
────────────────────┴────────────────────────────────────────┴────────────────────────┴────────────────────────

 REGLAS CLAVE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 • En { name } de getByRole  → SIEMPRE usá /regex/i.  Evita errores por espacios o mayúsculas.
 • { exact: true } solo aplica en getByText() y getByLabel(), NO dentro de { name } de getByRole.
 • /texto/i = la i al final ignora mayúsculas/minúsculas (case-insensitive).
 • /texto/  = busca la cadena tal cual, respeta mayúsculas.

 ⚠  TRAMPA DEL FALSO POSITIVO — toHaveAttribute con objeto
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 ❌  toHaveAttribute('href', { name: 'http://url.com', exact: true })
     El test puede PASAR aunque la URL sea diferente. Solo verifica que el atributo exista.

 ✅  toHaveAttribute('href', 'http://elementalselenium.com/')   ← string exacto
 ✅  toHaveAttribute('href', /elementalselenium/)               ← regex parcial

                 Si el test sigue pasando → está mal escrito. Usá string o regex.
